# Internationalization (i18n) Guide

This document explains how to use and contribute to the internationalization feature of the EmpowerH platform.

## Overview

EmpowerH uses [vue-i18n](https://vue-i18n.intlify.dev/) to provide multi-language support. Currently, the platform supports:

- **English (en)** 🇬🇧 - Default language
- **French (fr)** 🇫🇷 - For French-speaking African countries
- **Swahili (sw)** 🇰🇪 - For East African countries

## For Users

### Changing Language

1. Look for the language switcher in the top navigation bar (shows flag and language code)
2. Click on it to see available languages
3. Select your preferred language
4. Your choice will be saved and persist across sessions

## For Developers

### Using Translations in Components

#### In Template (Composition API)

```vue
<script setup>
import { useI18n } from 'vue-i18n'

const { t } = useI18n()
</script>

<template>
  <h1>{{ t('home.title') }}</h1>
  <p>{{ t('home.subtitle') }}</p>
</template>
```

#### In Template (Options API)

```vue
<script>
export default {
  setup() {
    const { t } = useI18n()
    
    return {
      t
    }
  }
}
</script>

<template>
  <h1>{{ t('home.title') }}</h1>
  <p>{{ t('home.subtitle') }}</p>
</template>
```

### Translation File Structure

Translation files are located in `src/i18n/locales/` and use JSON format:

```json
{
  "category": {
    "key": "Translation text",
    "nested": {
      "deepKey": "Deeply nested translation"
    }
  }
}
```

### Adding a New Language

1. **Create a new locale file:**
   ```bash
   touch src/i18n/locales/ar.json  # Example: Arabic
   ```

2. **Copy the English structure:**
   ```bash
   cp src/i18n/locales/en.json src/i18n/locales/ar.json
   ```

3. **Translate the values** (keep keys the same, translate only the values)

4. **Import in `src/i18n/index.js`:**
   ```javascript
   import ar from './locales/ar.json'
   
   const i18n = createI18n({
     // ...
     messages: {
       en,
       fr,
       sw,
       ar  // Add your new language
     }
   })
   ```

5. **Add to LanguageSwitcher** (`src/components/LanguageSwitcher.vue`):
   ```javascript
   const languages = [
     { code: 'en', name: 'English', flag: '🇬🇧' },
     { code: 'fr', name: 'Français', flag: '🇫🇷' },
     { code: 'sw', name: 'Kiswahili', flag: '🇰🇪' },
     { code: 'ar', name: 'العربية', flag: '🇸🇦' }  // Add here
   ]
   ```

### Translation Categories

The translation files are organized into the following categories:

- `nav` - Navigation menu items
- `home` - Home page content
- `about` - About page content
- `contact` - Contact page content
- `footer` - Footer content
- `events` - Events page content
- `blog` - Blog page content
- `auth` - Authentication forms (login/signup)
- `terms` - Terms and conditions
- `common` - Common UI elements (buttons, labels, etc.)

### Best Practices

1. **Use descriptive keys:**
   ```javascript
   // Good
   t('contact.submitButton')
   
   // Bad
   t('btn1')
   ```

2. **Keep translations organized:**
   - Group related translations under common categories
   - Use nested objects for complex pages

3. **Provide fallbacks:**
   - Always have English translations as the default
   - Use the same key structure across all language files

4. **Handle pluralization:**
   ```javascript
   // In locale file
   {
     "items": "no items | one item | {count} items"
   }
   
   // In component
   t('items', count)
   ```

5. **Handle parameters:**
   ```javascript
   // In locale file
   {
     "welcome": "Welcome, {name}!"
   }
   
   // In component
   t('welcome', { name: userName })
   ```

### Testing Translations

1. Run the development server:
   ```bash
   npm run dev
   ```

2. Navigate to the page you're testing

3. Use the language switcher to change languages

4. Verify all text is translated correctly

## Contributing Translations

We welcome contributions in any language! Here's how to contribute:

1. Fork the repository
2. Create a new branch for your translation
3. Add your locale file in `src/i18n/locales/`
4. Update `src/i18n/index.js` and `src/components/LanguageSwitcher.vue`
5. Test your translations
6. Submit a pull request

### Translation Guidelines

- **Be culturally sensitive:** Consider cultural context when translating
- **Maintain tone:** Keep the supportive and empowering tone of the platform
- **Technical accuracy:** Ensure technical terms are translated appropriately
- **Consistency:** Use consistent terminology throughout
- **Test thoroughly:** Make sure all pages display correctly in your language
